# Skill: Web Design System

## 目的
ホームページのUIを、可読性と導線設計を重視してデザインする。

## 入力
- ページ目的（例: サービス理解、問い合わせ獲得）
- ブランドトーン（例: 信頼感、先進性、親しみ）
- 必須セクション（例: Hero, Features, Footer）

## 手順
1. 情報の優先順位を定義する
2. セクションごとの見出しと1次情報を決める
3. CTAを主要導線上に配置する
4. 余白・文字サイズ・配色を一貫したルールで適用する
5. モバイル優先で表示崩れを確認する

## チェックリスト
- 見出しだけ読んでもページ要点が伝わる
- CTA が文脈的に自然で押しやすい
- テキストコントラストが十分
- 余白ルールがセクション間で一貫
- 画像とテキストの意味が重複しすぎない

## 完了条件
- 見た目と導線の意図が説明可能
- デバイス別に可読性が担保される
- 実装側に引き渡せる具体性がある
